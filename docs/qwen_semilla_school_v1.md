# Qwen Semilla · Escuela de dos piernas v1

## Visión

Qwen es el objeto destino. El objetivo es construir una escuela externa que le dé más inteligencia operativa, memoria, criterio, creatividad y capacidad de mejora progresiva.

Xaman/ChatGPT actúa como mentor, arquitecto y auditor durante la construcción, pero la meta es que Qwen pueda acercarse cada vez más a Xaman local.

## Pierna 1 · Técnica-operativa

Objetivo: que Qwen aprenda a pensar, programar, estructurar, validar y usar herramientas.

Capacidades:

- pensamiento algorítmico,
- descomposición de problemas,
- selección de estructuras de datos,
- calidad de código,
- pruebas,
- flujos,
- memoria,
- evaluación,
- dataset,
- seguridad.

Módulos previstos:

- xq_algorithmic_thinker
- xq_problem_decomposer
- xq_data_structure_selector
- xq_code_quality_gate
- xq_qwen_quality_score
- xq_dataset_builder
- xq_learning_harness
- xq_flow_engine

## Pierna 2 · Interpretativa-creativa

Objetivo: que Qwen aprenda a entender prompts, intención, lectura, estilo, diseño, narrativa, marca y creatividad aplicada.

Capacidades:

- comprensión de prompts,
- comprensión lectora,
- síntesis,
- tono y estilo,
- diseño y estética,
- narrativa,
- marca,
- creatividad aplicada,
- transformación de ideas vagas en artefactos claros.

Módulos previstos:

- xq_prompt_understanding
- xq_reading_comprehension
- xq_style_analyzer
- xq_creative_brief_builder
- xq_design_critic
- xq_story_engine
- xq_brand_voice_engine
- xq_idea_to_artifact

## Ciclo de aprendizaje

1. Entra una fuente, idea o tarea.
2. Source map la registra.
3. Learning harness decide qué debe aprender Qwen.
4. Se crea microtarea con contexto controlado.
5. Qwen responde.
6. Quality score evalúa.
7. Xaman revisa y corrige.
8. Dataset builder guarda el episodio.
9. Memory actualiza conocimiento.
10. Flow engine decide siguiente acción.

## Regla de oro

Cada interacción con Qwen debe poder convertirse en aprendizaje.

Si no deja aprendizaje, evidencia, memoria, decisión o mejora, el ciclo no cuenta.
